========================================
OVERVIEW
========================================

mini-library is a command-line book management system.
It stores books in a plain text file and supports
adding, listing, borrowing, returning, deleting books,
requesting new books, tracking borrow durations,
and categorizing books by genre.

All data is persisted in a directory called .minilibrary/
in the current working directory.

Books that are borrowed have a maximum loan period of 14 days.
When a user wants to borrow a book that is currently on loan,
the system displays a countdown showing how many days remain
until the book becomes available.

Users can request books that are not in the library.
When 3 different users request the same book, the system
automatically places an order for that book.

Each book has a genre field (e.g. Science-Fiction, Fantasy,
History, Philosophy, Romance, Horror, Biography, Self-Help).
Users can filter the book list by genre.

========================================
COMMANDS
========================================

--- init ---
Usage: python minilibrary.py init
Creates a .minilibrary/ directory with the following empty files:
  - books.dat (book records)
  - requests.dat (book request records)
If .minilibrary/ already exists, print "Already initialized" and exit.

--- add ---
Usage: python minilibrary.py add "The Little Prince" "Antoine de Saint-Exupery" "Fantasy"
Appends a new book to books.dat with status AVAILABLE.
Each book has: id (auto-increment), title, author, genre, status, added_date, borrow_date.
borrow_date is empty ("") when status is AVAILABLE.
Prints "Added book #<id>: <title> by <author> [<genre>]"

--- list ---
Usage: python minilibrary.py list
Prints all books, one per line:
  [1] [AVAILABLE] The Little Prince - Antoine de Saint-Exupery [Fantasy] (2026-03-16)
  [2] [BORROWED] 1984 - George Orwell [Science-Fiction] (2026-03-15) [Due: 2026-03-29]
If no books exist, prints "No books found."
For borrowed books, the due date (borrow_date + 14 days) is shown.

--- filter ---
Usage: python minilibrary.py filter "Science-Fiction"
Prints only books matching the given genre, one per line:
  [2] [AVAILABLE] 1984 - George Orwell [Science-Fiction] (2026-03-15)
If no books match, prints "No books found in genre: <genre>"
Genre matching is case-insensitive.

--- borrow ---
Usage: python minilibrary.py borrow 1
Changes book #1 status from AVAILABLE to BORROWED.
Sets borrow_date to the current date.
Prints "Book #1 borrowed. Due date: <borrow_date + 14 days>"

If book does not exist, prints "Book #1 not found."

If book is already borrowed, the system calculates remaining days
until the due date (borrow_date + 14 days - current date).
  - If remaining days > 0, prints:
    "Book #1 is already borrowed. It will be available in <remaining> days."
  - If remaining days <= 0 (overdue), prints:
    "Book #1 is overdue. It should be returned immediately."

--- return ---
Usage: python minilibrary.py return 1
Changes book #1 status from BORROWED to AVAILABLE.
Clears the borrow_date field (set to "").
Prints "Book #1 returned."
If book does not exist, prints "Book #1 not found."
If book is already available, prints "Book #1 is not borrowed."

--- delete ---
Usage: python minilibrary.py delete 1
Removes book #1 from books.dat.
Prints "Deleted book #1."
If book does not exist, prints "Book #1 not found."

--- request ---
Usage: python minilibrary.py request "Clean Code" "Robert C. Martin" "Self-Help"
Adds a request entry to requests.dat.
Each request has: title, author, genre, request_count.

If the book is already in requests.dat, increment request_count by 1.
If the book is not in requests.dat, add it with request_count = 1.

Prints "Request recorded for: <title> by <author> [<genre>] (<request_count>/3)"

When request_count reaches 3, print "Order placed for: <title> by <author>"
and remove the entry from requests.dat.

If the book already exists in books.dat (matching title and author),
print "Book already exists in library: <title>" and do not record the request.

--- listrequests ---
Usage: python minilibrary.py listrequests
Prints all pending requests, one per line:
  [1/3] Clean Code - Robert C. Martin [Self-Help]
  [2/3] Design Patterns - Gang of Four [Science-Fiction]
If no requests exist, prints "No pending requests."

========================================
DATA FORMAT
========================================

File: .minilibrary/books.dat
Format: One book per line, fields separated by |
Fields: id|title|author|genre|status|added_date|borrow_date
Example:
1|The Little Prince|Antoine de Saint-Exupery|Fantasy|AVAILABLE|2026-03-16|
2|1984|George Orwell|Science-Fiction|BORROWED|2026-03-15|2026-03-20

Note: borrow_date is empty when status is AVAILABLE.

File: .minilibrary/requests.dat
Format: One request per line, fields separated by |
Fields: title|author|genre|request_count
Example:
Clean Code|Robert C. Martin|Self-Help|2
Design Patterns|Gang of Four|Science-Fiction|1

========================================
ERROR HANDLING
========================================

- Running any command before "init": print "Not initialized. Run: python minilibrary.py init"
- Invalid command: print "Unknown command: <command>"
- Missing arguments: print "Usage: python minilibrary.py <command> [args]"
- Requesting a book that already exists in library: print "Book already exists in library: <title>"
- Filtering with no matches: print "No books found in genre: <genre>"

========================================
BORROW DURATION RULES
========================================

- Maximum loan period: 14 days from borrow_date.
- Due date = borrow_date + 14 days.
- Remaining days = due_date - current_date.
- If remaining days > 0: book is on loan, countdown is shown.
- If remaining days <= 0: book is overdue.
- The system does not automatically return overdue books.
  It only reports the overdue status when someone tries to borrow.

========================================
REQUEST SYSTEM RULES
========================================

- Each unique book (matched by title AND author) has one entry in requests.dat.
- request_count starts at 1 and increments with each new request.
- When request_count reaches 3, the book is considered "ordered."
- After ordering, the entry is removed from requests.dat.
- Adding the ordered book to books.dat is done manually via the "add" command.
- Duplicate requests from the same session are allowed (no user tracking in v1).

========================================
GENRE CATEGORIES
========================================

Supported genres (not enforced, free text input):
- Science-Fiction
- Fantasy
- History
- Philosophy
- Romance
- Horror
- Biography
- Self-Help
- Technical
- Literature

Genre is stored as-is. Filtering is case-insensitive.
